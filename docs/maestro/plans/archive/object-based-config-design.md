# Design Document: Object-Based Configuration in Deployment Script

The `deploy-models.py` script currently generates `docker-compose.yml` and `litellm_config.yaml` using string concatenation. This is brittle and difficult to maintain as the configuration grows more complex. This design document outlines the transition to using Python dictionaries and the `yaml` library for configuration management.

## Proposed Changes

### Configuration Objects
The script will define Python dictionaries to represent the structure of the YAML files.
- `litellm_config`: A dictionary with a `model_list` key containing model definitions.
- `compose_dict`: A dictionary with `services` and `volumes` keys representing the Docker Compose stack.

### Dynamic Service Generation
vLLM services will be generated by creating individual dictionaries for each model and adding them to the `services` dictionary. This makes it easier to manage per-model settings like ports, volumes, and GPU allocations.

### Static Services and Volumes
The `db` and `litellm` services, as well as the top-level `volumes` block, will be defined as static dictionary structures. The `litellm` service's `depends_on` list will be dynamically updated to include all generated vLLM services and the `db` service.

### YAML Serialization
The `yaml.dump` function from the `PyYAML` library will be used to serialize these dictionary objects into valid YAML files. This ensures correct formatting, indentation, and escaping, which are difficult to manage manually with string templates.

## Benefits
- Improved maintainability and readability.
- Reduced risk of YAML syntax errors.
- Easier to extend with new configuration options or sidecar services.
- Clean separation between logic (model discovery) and representation (YAML generation).
